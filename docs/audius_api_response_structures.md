# Audius API Response Structures

These are the response shapes the extension actually reads. They reflect the current
`https://api.audius.co/v1` responses, which now include `mirrors` arrays on every image
object. For the authoritative spec, fetch [`api.audius.co/v1/swagger.yaml`](https://api.audius.co/v1/swagger.yaml).

> The service layer (`services/audiusApi.js`) unwraps the top-level `{ "data": ... }` envelope,
> so the types below describe what `fetchApi` returns to callers.

## Image objects (artwork / profile_picture / cover_photo)

```ts
interface ImageObject {
    "150x150"?: string;
    "480x480"?: string;
    "1000x1000"?: string;
    "640x"?: string;     // cover_photo
    "2000x"?: string;    // cover_photo
    mirrors?: string[];  // alternate content-node origins for the same path
}
```

`mirrors` is critical — when a content node is unreachable, the extension swaps the URL's
origin with each entry of `mirrors` and retries. Don't discard it.

## Playlist (`GET /v1/playlists/{id}` or `GET /v1/playlists/by_permalink/{handle}/{slug}`)

The list endpoints return an **array**; the by_permalink endpoint also returns an array. The
service layer normalizes to a single playlist via `[0]`.

```ts
interface Playlist {
    id: string;
    permalink: string;
    playlist_name: string;
    description: string;
    is_album: boolean;
    is_image_autogenerated: boolean;
    artwork: ImageObject;
    track_count: number;
    repost_count: number;
    favorite_count: number;
    total_play_count: number;
    user: User;
    playlist_contents: Array<{ metadata_timestamp: number; timestamp: number; track_id: string }>;
    access: { stream: boolean; download: boolean };
    upc: string | null;
    ddex_app: string | null;
}
```

Use `GET /v1/playlists/{id}/tracks` to get the tracks for a playlist (the service returns
an array directly).

## Notes

- All requests include `app_name=snag_chrome_extension`. If a user provides an API key
  via the options page, the extension also sends `x-api-key: <key>`.
- The `data` envelope is always present; the service layer unwraps it.
- `/v1/playlists/by_permalink/{handle}/{slug}` takes the album/playlist *slug* — not the
  full `handle/album/slug` URL path. The service layer extracts the slug for you.
- `discoveryprovider.audius.co` is deprecated — only use `https://api.audius.co`.
